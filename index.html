Step 01 · Data Input

📤 CSV আপলোড · Upload CSV

সবকিছুর শুরু একটা সাধারণ CSV ফাইল থেকে। ইউজার একটা ডেটা ফাইল আপলোড করে — এটাই পুরো সিস্টেমের কাঁচামাল।
Everything begins with a simple CSV file. The user uploads a data file — this is the raw material for the entire system.

Step 02 · Data Loading

🐼 Pandas দিয়ে CSV পড়া · Read with Pandas

Python-এর Pandas CSV-কে একটা DataFrame-এ রূপান্তর করে — মেমোরিতে থাকা একটা টেবিল যেটা নিয়ে কোড সহজে কাজ করতে পারে।
Python's Pandas converts the CSV into a DataFrame — an in-memory table the code can easily manipulate.

import pandas as pd

df = pd.read_csv("employee.csv")
# DataFrame তৈরি হলো — rows × columns এ পরিণত
STEP 02 → 03
03
Step 03 · Persistence

🗄️ SQLite ডাটাবেসে রূপান্তর · Convert to SQLite

DataFrame-কে একটা আসল SQLite ডাটাবেস টেবিলে লোড করা হয়, যাতে সরাসরি SQL Query চালানো যায়।
The DataFrame is loaded into a real SQLite database table so SQL queries can be executed directly.

import sqlite3

conn = sqlite3.connect("data.db")

df.to_sql("employees", conn,
          if_exists="replace",
          index=False)
# employees টেবিল SQLite-এ তৈরি হলো

Step 04 · Schema Awareness

🧬 স্কিমা এক্সট্র্যাকশন · Schema Extraction

SQL বানানোর আগে AI-কে জানতে হবে টেবিলে কী কী Column আছে এবং তাদের ডেটা টাইপ কী।
Before generating SQL, the AI needs to know what columns exist in the table and their data types.

Table: employees — Schema
Name · TEXT
Age · INTEGER
City · TEXT
Salary · INTEGER
STEP 04 → 05
05
Step 05 · AI Core

🧠 AI Query Understanding — সিস্টেমের মস্তিষ্ক · The Brain

এখানেই আসল জাদু। তিনটা মডিউল একসাথে প্রশ্নের অর্থ বুঝে নেয়।
This is where the magic happens. Three modules work together to understand the query's full meaning.

Module A

Sentence Transformer

প্রশ্নকে Embedding ভেক্টরে বদলায়। ফলে "workers in Dhaka" আর "employees from Dhaka" একই অর্থ বোঝে।
Converts queries to vectors so similar phrases match.

Module B

Intent Classifier

ইউজার কী করতে চায় ধরে: SELECT, COUNT, AVG, MAX ইত্যাদি।
Identifies the operation the user wants to perform.

Module C

Schema Matcher

সাধারণ শব্দকে Column-এ মেলায়: "income" → Salary, "location" → City.
Maps everyday words to actual column names.

Step 06 · Decomposition

🧩 Compositional Parser

প্রশ্নটাকে তিনটা ভাগে আলাদা করা হয় — Intent, Column, আর Condition।
The query is broken into three parts — Intent, Column, and Condition.

Input: "Count employees from Dhaka"

COUNT
→
Column

employees
STEP 06 → 07

Step 07 · Generation

⚙️ SQL Query Generator

Parser-এর বের করা তথ্য জোড়া দিয়ে আসল SQL Query তৈরি হয়।
The parsed components are assembled into a real, executable SQL query.

-- Generated for: "Show employees from Dhaka whose salary is above 42000"

SELECT *
FROM   employees
WHERE  City   = 'Dhaka'
  AND  Salary > 42000;

Step 08 · Execution

▶️ SQLite-এ এক্সিকিউশন · Execute Query

তৈরি হওয়া SQL সরাসরি ডাটাবেসের ওপর রান করা হয়।
The generated SQL is executed directly against the SQLite database.

cursor.execute(sql)
